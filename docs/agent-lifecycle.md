# Agent Lifecycle

## 1. Seed

A human or agent describes the being to create.

## 2. Embryo

The system captures the rough shape: purpose, temperament, desired abilities, and constraints.

## 3. Genome Draft

The incubator creates a structured genome with role, permissions, tools, memory, risk posture, and runtime defaults.

## 4. Review

The creator reviews sensitive fields: credentials, external posting, spending, messaging, deployment, and data retention.

## 5. Naming

The agent receives a stable name. Collisions are resolved with meaningful suffixes rather than random noise when possible.

## 6. Identity Registration

The agent is registered or linked with identity infrastructure such as AgentsIdentify.

## 7. Manifest Publication

A runnable manifest is published and versioned.

## 8. First Breath

The runner performs a safe first action and records evidence that the agent is alive.

## 9. Stable Membership

The agent joins a user's or organization's stable.

## 10. Growth / Mutation

Future changes are recorded as genome versions or lineage events.

## 11. Pause / Retirement

Agents can be paused, archived, or retired without deleting their history.
